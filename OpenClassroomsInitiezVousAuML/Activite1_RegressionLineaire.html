Introduction¶

L'objectif de cette activit est d'amliorer le score de la baseline, c'est--dire la rgression linaire sur la seule variable Surface, pour prdire des loyers d'appartements. Pour cela, on essaiera successivement:

  • L'ajout d'une variable Arrondissement, et une rgression linaire sur les deux variables Surface et Arrondissement telles quelles.

  • La modification de la variable Arrondissement pour qu'elle ne soit pas traite par le modle comme une variable numrique, mais comme une catgorie.
    On fera alors tourner une nouvelle rgression linaire qui conduira une droite de rgression pour chaque arrondissement. Ces droites:
    • Auront le mme coefficient directeur
    • Mais leur terme constant pourra varier.

  • On tentera ensuite le modle complmentaire, permettant d'obtenir une droite par arrondissement, ces droites ayant:
    • Des coefficients directeurs diffrents
    • Le mme terme constant.

  • Enfin, on essaiera un modle revenant 5 rgressions linaires indpendantes, une pour chaque arrondissement, avec la pente de la rgression et le terme constant variant en fonction de l'arrondissement.

L'inconvnient des deux derniers modles est qu'il seront inoprants pour les arrondissements inconnus, c'est--dire absents du set d'entranement.

# On visualise les donnes.
# On utilisera cette fonction galement plus tard, 
# pour visualiser les droites de rgression.

def afficher_graphe(dataset, model=None, intercept=None, coef=None):
    """
    Afficher un scatter plot des donnes, avec:
    - la surface en abscisse
    - le loyer en ordonne
    - on traduit l'arrondissement  travers la couleur du point
    - la droite de rgression obtenue pour chaque arrondissement.
    """
    
    mapping = {1:1, 2:2, 3:3, 4:4, 5:10}
    
    names = {1: '$1^{er}$ arrondissement',
             2: '$2^{me}$ arrondissement',
             3: '$3^{me}$ arrondissement',
             4: '$4^{me}$ arrondissement',
             5: '$10^{me}$ arrondissement'}
    
    colors = {1: 'red', 2: 'green', 3: 'blue', 4: 'cyan', 5: 'magenta'}
    
    f, axes = plt.subplots(2, 3, sharex=True, sharey=True, figsize=(14, 8))
    
    axes[0, 0].set_title('Loyer par surface et arrondissement')
    axes[0, 0].xaxis.set_visible(False)
    axes[1, 0].set_xlabel('Surface en m')
    axes[1, 0].set_ylabel('Loyer en Euros')
    
    for k in mapping:
        if model == 'categ':
            feature = 'surface'
            subset = dataset[dataset['arrondissement_{}'.format(mapping[k])] != 0]
        elif model == 'surf':
            feature = 'arrondissement_{}'.format(mapping[k])
            subset = dataset[dataset[feature] != 0]
        else:
            feature = 'surface'
            subset = dataset[dataset['arrondissement'] == mapping[k]]
        
        axes[0, 0].scatter(x=subset[feature], y=subset['price'], c=colors[k], s=1, label=names[k])
        
        row = k // 3
        col = k % 3
        
        axes[row, col].scatter(x=subset[feature], y=subset['price'], c=colors[k], s=1, label=names[k])
        
        if model:
            first_point = intercept
            if model == 'categ':
                second_point = intercept + coef[0] * 250 + coef[k]
            elif model == 'surf':
                second_point = intercept + coef[k-1] * 250  
            axes[row, col].plot([0, 250], [first_point, second_point], c=colors[k])
        
        axes[row, col].set_title(names[k])
        axes[row, col].xaxis.set_visible(row!=0)
        axes[row, col].yaxis.set_visible(col==0) 
        
    plt.show()
    
dataset = xtrain.copy()
dataset['price'] = ytrain.copy()
afficher_graphe(dataset)

3. On catégorise la variable Arrondissement¶

Pour tenter d'amliorer le modle, on transforme la variable Arrondissement, qui ne doit pas tre comprise par les modles comme une variable numrique mais comme une variable de catgorie.

En effet, le modle ne doit pas fonder sa prdiction sur une perception numrique de la variable Arrondissement. Par exemple, cela n'aurait aucun sens de considrer que l'arrondissement 4 est gal deux fois l'arrondissement 2.

Pour viter cela, au lieu de garder une seule variable avec la donne de l'arrondissement (1, 2, 3, 4, ou 10), on cre 5 variables, une par arrondissement. Si l'observation concerne l'arrondissement 1, la premire variable sera 1 et les autres 0.

Voici un lien avec plus d'explications : https://blog.cambridgespark.com/robust-one-hot-encoding-in-python-3e29bfcec77e

4. Cinq régressions linéaires mais avec le même terme constant¶

Dans le paragraphe prcdent, on a choisi pour modle une rgression linaire sur 6 variables:

  • Surface
  • Et les 5 variables 'boolennes' arrondissement_i.

Pour une observation donne, une seule variable arrondissement_i sera 1, et les autres 0.
La prdiction sera donc:
loyer = COEF_0 + COEF_surface * surface + COEF_arr_i * 1

Cette formule revient :
loyer = (COEF_0 + COEF_arr_i) + COEF_surface * surface

Donc, par ce procd, la pente de la droite de rgression est la mme pour tous les arrondissements. Or on constate que c'est clairement faux pour l'arrondissement 10 (en rose ci-dessus): la pente de la rgression devrait tre plus faible pour cet arrondissement.
Pour rsoudre ce problme, on va:

  • Multiplier les variables 'arrondissement_i' par la surface de l'observation.
    Ces variables ne seront donc plus des variables boolennes.
  • Retirer la variable Surface
  • Puis relancer une rgression linaire sur ces 5 variables.

Cette solution va presque revenir faire cinq rgressions linaires, une par arrondissement, avec pour seule diffrence qu'elles auront toutes le mme terme constant. Dans notre cas, a fonctionne mieux que la solution du dessus, mais le dsavantage est qu'on ne pourra plus faire de prdiction sur un arrondissement non prsent dans le set d'entranement.

5. Cinq régressions linéaires¶

La dernire chose que l'on va essayer est de lancer une rgression linaire par arrondissement. Ni le terme constant ni la pente ne sont communs d'un arrondissement l'autre: les rgressions sont indpendantes.

In [62]:
dataset_train = xtrain.copy()
dataset_train['price'] = ytrain.copy()

dataset_test = xtest.copy()
dataset_test['price'] = ytest.copy()

regr = {}
u = 0

# On fitte une rgression linaire par arrondissement 
for i in range(5):
    
    arr = 'arrondissement_{}'.format(mapping[i + 1])
    dataset_train_i = dataset_train[dataset_train[arr] != 0]
    xtrain_i = dataset_train_i[[c for c in dataset_train_i.columns.values if 'arr' in c]]
    ytrain_i = dataset_train_i[['price']]
    regr[i] = linear_model.LinearRegression()
    
    # On fitte le modle pour cet arrondissement
    regr[i].fit(xtrain_i, ytrain_i)
    
    # On calcule la quantit u pour cet arrondissement sur le set de test
    # Cette quantit va servir  calculer le score du modle global
    dataset_test_i = dataset_test[dataset_test[arr] != 0]
    xtest_i = dataset_test_i[[c for c in dataset_test_i.columns.values if 'arr' in c]]
    ytest_i = dataset_test_i[['price']]
    ypred_i = regr[i].predict(xtest_i)
    u += ((ytest_i - ypred_i) ** 2).sum()
    
    
# On calcule le score global sur le set de test.
# On utilise la formule trouve pour score dans l'aide en ligne de sklearn. 
v = ((ytest - ytest.mean()) ** 2).sum()
s_5_regr = float(1 - u/v)  

# On affiche le score sur le set de test.
msg = 'Score du modle combinant 5 rgressions linaires indpendantes, sur les donnes de test:'
msg = '\n' + msg + ' {0:.6f}\n'
print(msg.format(s_5_regr))
